# 📘 คู่มือการตั้งค่า Google OAuth 2.0 Client ID สำหรับ OrgChat-AI
คู่มือนี้เขียนขึ้นเป็นภาษาไทยเพื่ออธิบายขั้นตอนการรับค่าสำหรับระบบล็อกอินด้วย Google (Sign in with Google) ตั้งแต่เริ่มต้นจากศูนย์ (ไม่มีโปรเจกต์ใด ๆ บน Google Cloud) เพื่อให้คุณสามารถทำตามหรือส่งมอบให้ผู้อื่นทำตามได้อย่างถูกต้องและง่ายดายที่สุด

---

## 🛠️ ขั้นตอนที่ 1: ลงทะเบียนโปรเจกต์ใหม่บน Google Cloud (Create a New Project)
1. เข้าไปที่ลิงก์ [Google Cloud Console](https://console.cloud.google.com/) 
2. เข้าสู่ระบบด้วยบัญชี Google ขององค์กร หรือบัญชี Gmail ทั่วไป
3. บริเวณแถบเมนูด้านบนสุด (ข้างโลโก้ **Google Cloud**) ให้คลิกปุ่ม **"Select a project"** หรือชื่อโปรเจกต์เดิมที่มีอยู่
4. หน้าต่างย่อยจะปรากฏขึ้น ให้คลิกปุ่ม **"New Project"** (ที่มุมขวาบนของหน้าต่างนั้น)
5. กรอกข้อมูลดังนี้:
   * **Project name**: ตั้งชื่อที่เข้าใจง่าย เช่น `OrgChat AI Authentication`
   * **Organization**: เลือกหน่วยงานของคุณ (หากเป็นบัญชีทั่วไป ให้ปล่อยเป็น *No organization*)
6. คลิกปุ่ม **"Create"**
7. รอระบบสร้างโปรเจกต์สักครู่ (จะมีกระดิ่งแจ้งเตือนขวาบน) เมื่อสร้างเสร็จแล้ว ให้คลิกเลือกสลับโปรเจกต์ให้เป็นโปรเจกต์ที่เพิ่งสร้างใหม่นี้

---

## 🛠️ ขั้นตอนที่ 2: ตั้งค่าหน้าจอขออนุญาต (Configure OAuth Consent Screen)
*ก่อนที่ Google จะอนุญาตให้สร้างรหัสผ่านเพื่อเชื่อมต่อระบบได้ เราจำเป็นต้องสร้างหน้าร้องขอความยินยอม (หน้าต่างที่จะเด้งขึ้นมาถามผู้ใช้เวลาล็อกอิน) เสียก่อน*

1. คลิกปุ่มขีดสามขีด (Hamburger Menu) ที่มุมซ้ายบน เพื่อเปิดแถบเครื่องมือหลัก
2. ไปที่เมนู **APIs & Services** ➡️ **OAuth consent screen**
3. เลือกประเภทผู้ใช้งานหลัก (**User Type**):
   * **External**: เลือกตัวเลือกนี้ เพื่อเปิดให้บัญชี Gmail ทั่วไป หรือบัญชีองค์กรใด ๆ สามารถล็อกอินเข้าสู่ระบบได้จริง
4. คลิกปุ่ม **"Create"**
5. กรอกข้อมูลแอปพลิเคชัน (กรอกเฉพาะช่องที่จำเป็น):
   * **App name**: พิมพ์ชื่อระบบของคุณ เช่น `OrgChat-AI`
   * **User support email**: เลือกอีเมลของคุณจากเมนู Dropdown
   * **Developer contact information**: กรอกอีเมลของคุณเพื่อรับการแจ้งเตือนจาก Google
6. คลิกปุ่ม **"SAVE AND CONTINUE"**
7. ในหน้าถัดไปคือ **Scopes** และ **Test users** ให้เลื่อนลงไปด้านล่างสุดแล้วกด **"SAVE AND CONTINUE"** ข้ามไปได้ทันทีโดยไม่ต้องตั้งค่าใด ๆ
8. เมื่อกลับมาที่หน้าหลักของ **OAuth consent screen** ให้คลิกปุ่ม **"Publish App"** (ใต้ Publishing status) และกดยืนยันเพื่อให้แอปพลิเคชันใช้งานได้จริงสู่สาธารณะ

---

## 🛠️ ขั้นตอนที่ 3: สร้าง Credentials (OAuth 2.0 Client ID)
1. ในแถบเมนูด้านซ้าย คลิกเลือกเมนู **Credentials**
2. คลิกปุ่ม **"+ Create Credentials"** ที่อยู่ด้านบนของหน้าจอ แล้วเลือกตัวเลือก **"OAuth client ID"**
3. กรอกข้อมูลและตั้งค่าที่สำคัญมากดังนี้:
   * **Application type**: เลือกเป็น **"Web application"**
   * **Name**: ตั้งชื่อรหัสเชื่อมต่อนี้ เช่น `OrgChat Web OAuth Client`
   * **Authorized JavaScript origins** (โดเมนหลักที่อนุญาตให้รันสคริปต์):
     * คลิกปุ่ม **"+ ADD URI"**
     * ใส่โดเมนหลักของคุณ (ห้ามใส่เครื่องหมาย `/` ปิดท้ายเด็ดขาด!) เช่น:
       * `https://openchat.sbs` (โดเมนบน Cloud)
       * `http://localhost:5005` (สำหรับเปิดทดสอบบนเครื่องตนเอง)
   * **Authorized redirect URIs** (ลิงก์ปลายทางขากลับที่อนุญาตหลังจากล็อกอินสำเร็จ):
     * คลิกปุ่ม **"+ ADD URI"**
     * กรอกลิงก์ปลายทางให้ตรงกับที่ระบบตั้งไว้:
       * `https://openchat.sbs/api/auth/google/callback` (โดเมนบน Cloud)
       * `http://localhost:5005/api/auth/google/callback` (สำหรับเปิดทดสอบบนเครื่องตนเอง)
4. คลิกปุ่ม **"Create"** ที่ด้านล่างสุด

---

## 🛠️ ขั้นตอนที่ 4: คัดลอกค่าไปใส่ในไฟล์ `.env` ของระบบ
เมื่อกดสร้างสำเร็จ ระบบจะแสดงหน้าต่างแจ้งเตือน **"OAuth client created"** ให้คัดลอกรหัสเหล่านั้นมาวางลงในไฟล์ `.env` ของคุณ:

```env
# 1. คัดลอก Client ID ทั้งหมด มาใส่ในตัวแปรทั้งสองตัวนี้ (ใช้ค่าเดียวกัน)
GOOGLE_OAUTH2_CLIENT_ID=38422568920-xxxxxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_ID=38422568920-xxxxxxxx.apps.googleusercontent.com

# 2. คัดลอก Client Secret มาใส่ในตัวแปรนี้
GOOGLE_OAUTH2_CLIENT_SECRET=GOCSPX-xxxxxxxxxxxxxxxxxxxxxxxx

# 3. กำหนดลิงก์ขากลับให้ตรงกับที่ระบุไว้ในขั้นตอนที่ 3
OAUTH2_REDIRECT_URI=https://openchat.sbs/api/auth/google/callback
```
